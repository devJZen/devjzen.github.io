### 날짜: JS에서 제공하는 메서드

> 언제까지 getDate만 쓸 것인가?

js 메서드 공부를 하다가 tolSOString()을 알게됐다.

```js

const today = new Date();
const splitT = today.toISOString().split("T");
console.log(splitT.[0]);

```

ISO 기준이기에 시간도 표준이다. 시차를 KST에 맞춰줘야 된다.
T를 기준으로 나누면 날짜와 시간을 분리해서 가져오기 좋다.
toLocaleString을 통해서 시차를 보정할 수 있고 또는 국제 표준에서 KST를 계산하여 보정할 수 있다.

ref: [w3school](https://www.w3schools.com/jsref/jsref_toisostring.asp), [coreui](https://coreui.io/answers/how-to-format-date-as-yyyy-mm-dd-in-javascript/)

#### 날짜: 기존 방식

```js

const today = new Date();
let day = today.getDate();
let month = today.getMonth();
let year = today.getYear();
console.log(day + "-" + month + "-" + year);

```

또는

```js

function ymdformatter();{
//year+month+day
}
```

구현이 매우 간단하다. 그러나 반복되면 포매터를 만들어서 그냥 공통으로 쓰고 가져와서 쓰도록 한다.

### const와 let, var의 차이

> 재 할당 아니던가?

let은 선언 후 나중에 값 할당이 되고 const는 안된다.
프론트는 신경쓰기 싫어서, sql문법과 비슷하다는 이유로, 그냥 편하니까 많이 썼는데 이러면 안된다고 해서 확실하게 구분해서 쓰려고 한다.

ref: [reddit](https://www.reddit.com/r/learnjavascript/comments/qtwuc3/what_is_the_difference_between_var_const_and_let/?tl=ko)

#### Scope와 재사용에 대하여

var를 쓰면 예측하기 어렵기에 const와 let을 명확하게 구분하여 사용하도록 한다.
표준에 가까운 코딩은 const다.
